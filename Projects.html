Projects

My goal with any project, personal or professional, is to enhance the skills I have currently and learn new ones. I imagine I have a tool box containing every skill I have learned throughout my college and professional career. I constantly am trying to add new tools to the box and hone the ones that are already in there.

Below are a few of the projects that I have either completed or am currntly working on. The projects themselves were completed in WPI courses, were personal projects, or were applications where I applied my skills to solve a problem in my everyday life. Projects that included some form of software will have a link to a GitHub repository where the code can be viewed, as well as any other supporting documents.

RBE: 3001

WPI structures the courses such that the beginning ones introduce mechanical, electrical and software skills while the later courses focus on less broad skills within these branches. One of the more advanced projects I worked on was using a 3 DOF arm to sort colored balls. We had to derive the forward and inverse kinematics for the arm to accomplish this task. All the code was done in MATLab which on its own was a bit of a challenge being that I had never done such a major project in that format. Despite this I like to say this is my favorite course that I took while attending WPI due to the challenge it posed and how well the theory transferred to practice in the lab.

RBE: 3002

Following this was the introduction to ROS. The goal of this project was to navigate a maze using A* and SLAM for localization. For this we used turtle bots with lidar. To start, everything was done in SIM then was migrated to a physical robot. This demonstrated that this transition is rarely seamless. The first challenge of the project was figuring out how to map the unknown area. We accomplished this by approaching the largest unknown area or frontier and reassessing the area until there were no longer unknown areas or frontiers. Once this was completed the robot was placed in a random location and needed to localize itself then drive to a desired position.

RBE: 450X

The latest project I completed was using a 2 DOF arm and a camera to find an ideal grasp for an object. The camera returned RGB D data in the form of a point cloud. This was again done using ROS 2 and was only in simulation. Once we had the point cloud, the top layer of the object was separated and from here an optimal grasp was derived. When I took it was in the experimental stage, and had never been taught with the use of ROS 2. For me this was my first time using ROS 2 and I not only became familiar with point clouds, edge detection and grasp derivation but also how to convert libraries meant for ROS 1 to ROS 2.

Major Qualifying Project

WPI requires every student to complete an MQP or major qualifying project. This is a project that takes place over the course of a year and is directly related to your major. For my project two other WPI students and I are working with Flexxbotics to better integrate a Universal Robot with a HAAS CNC machine using ROS to resolve alarms on the machine during lights out operation. ROS is also being used to aggregate data from both systems and report run records to the cloud for analysis.
