MQP: Autonomous CNC alarm resolution

overview

WPI requires every student to complete an MQP or major qualifying project. This is a project that takes place over the course of a year and is directly related to your major. For my project two other WPI students and I are working with Flexxbotics to better integrate a Universal Robot with a HAAS CNC machine using ROS to resolve alarms on the machine during lights out operation. ROS is also being used to aggregate data from both systems and report run records to the cloud for analysis.

How it works

This project is currently still in progress, but the current plan is to use a universal robot along side a flexxbotics edge device that interfaces with the robot via ROS and moveit. When an alarm is present on the CNC a secondary ros device is responsible for controlling the motion of the robot using moveit. During CNC operation and alarm resolution the Flexx device is monitoring all run time data and agrigates it into a run record. This data is obtained via the universal robots ROS driver.
